## 5. Data ownership map

**Профиль:** базовый `users` (identity, telegram) → `user_profiles` (nutrition
profile, SoT для целей/диет/аллергий) → `family_members.nutrition_profile`
(virtual members). Onboarding пишет тот же `UserProfile`. Health/wellness goals —
`user_profiles.pro_data` + `progress.NutritionTarget`.

**Меню:** `family_menu_selections.menu_data` (selected menu + generated
variants) → daily plan = recipe-slot план в том же JSONB; meal check-ins =
`meal_checkins`; nutrition aggregate = `recipes.nutrition_*` →
`nutrition.plan_aggregator`.

**Рецепты:** `recipes` row + `recipe_ingredients` rows (SoT) ↔ `recipes.ingredients`
JSONB (legacy/display, синхронизируется `recipe_storage`).

**Покупки:** `family_shopping_lists.items` JSONB — menu-generated (source=menu) +
manual (source=manual); merge в pantry через `pantry_shopping`.

**Остатки:** `family_pantry_items` — manual + receipt OCR + voice + checked
shopping items.

**Уведомления:** `care_settings` (care mode, quiet hours) + `user_notification_settings`
(buy/cook reminders) → `notification_scheduler` → Telegram bot.

**Подписка:** `subscription_plans` (catalog) + `user_subscriptions` (plan/status/
usage) + admin overrides (`metadata_json`) + AMA wallet.

## 8. Безопасная стратегия удаления старого

**Stage 1 — Consolidate (этот коммит):** единые normalizers + подключённые
write-paths + route registry + health audit + пометка legacy. Ничего не удалено.

**Stage 2 — Freeze legacy (следующий шаг):** добавить `# DEPRECATED (Legacy
Cleanup V2)` комментарии в legacy-сервисы; тест, что новые сервисы не импортируют
`menu_ai_legacy` напрямую (кроме существующей fallback-точки); запрет новых
импортов legacy frontend компонентов вне fallback.

**Stage 3 — Remove legacy (Legacy Cleanup V2, отдельный коммит):** удалить только
подтверждённо неиспользуемое после grep/import-check/build. Кандидаты — раздел 7.

### Legacy Cleanup V2 candidates (чек-лист)

- [ ] `services/menu_ai_legacy.py` — после переноса leftovers/fallback в recipe-DB путь.
- [ ] `services/shopping_category_migration.py` — после подтверждения, что миграция применена в prod.
- [ ] legacy frontend компоненты с 2026-заменой (раздел 7).
- [ ] deprecated route-папки — только после удаления входящих ссылок и middleware.
- [ ] `routers/recipes._normalized_ingredient_name` → `normalization.ingredients`.
