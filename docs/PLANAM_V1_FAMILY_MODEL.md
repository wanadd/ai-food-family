# Главный вопрос

```text
Почему семье удобнее пользоваться PlanAm вместе, а не по отдельности?
```

**Ответ V1:**

Семья получает **одно меню, один список покупок и одну картину здоровья** — без пересылок в чатах «что купить» и «что готовим». Список покупок — **единый семейный центр**: ингредиенты из меню, бытовые товары, детские и зоотовары в одном месте. PlanAm учитывает аллергии и предпочтения каждого **до** того, как кто-то открыл холодильник.

---

# 1. Роли в семье

| Роль | Кто | Права V1 |
|------|-----|----------|
| **admin** | Создатель семьи, обычно один из родителей | Приглашения, удаление участников, переименование семьи, редактирование профилей детей |
| **adult** | Супруг, родитель, бабушка (Telegram-аккаунт) | Просмотр меню/покупок, отметка «приготовил», свои уведомления |
| **child** | Ребёнок (часто virtual member без Telegram) | Учитывается в меню; профиль ведёт admin |

**Virtual member** — участник без Telegram (ребёнок, пожилой родственник). Admin заполняет nutrition profile от его имени.

# 2. Ежедневное использование семьёй

## Утро (будни)

| Шаг | Кто | Действие | Экран |
|-----|-----|----------|-------|
| 1 | Любой | Открывает PlanAm | ПланАм `/` |
| 2 | Все видят | Hero: ближайший приём пищи (динамически) | ПланАм |
| 3 | Admin | Видит «7 товаров купить» | Статус 🛒 |
| 4 | Супруг | Идёт в магазин, открывает Покупки | `/home/shopping` |
| 5 | Супруг | Отмечает купленное | Покупки |
| 6 | Готовящий | «Приготовить» на Сегодня | `/plan/today` |

## Вечер

| Шаг | Кто | Действие |
|-----|-----|----------|
| 1 | Готовящий | Отмечает приём пищи (outcome sheet) |
| 2 | Система | Обновляет остатки, калории семьи |
| 3 | Все | На следующее утро — новый hero на ПланАм |

# 3. Добавление родственников

## Сценарий A: Супруг с Telegram

| Шаг | Действие |
|-----|----------|
| 1 | Admin → Профиль → Семья |
| 2 | «+ Добавить человека» → «Пригласить в Telegram» |
| 3 | Ввод телефона или share link |
| 4 | Супруг получает invite в Telegram bot |
| 5 | Супруг принимает → появляется в family mode |
| 6 | App context переключается на family; общее меню и покупки |

## Сценарий B: Ребёнок без Telegram

| Шаг | Действие |
|-----|----------|
| 1 | Admin → Семья → «Добавить без Telegram» |
| 2 | Имя, возраст (месяцы), цель питания |
| 3 | Аллергии, ограничения, любимое/нелюбимое |
| 4 | Согласие (guardian consent для child) |
| 5 | Virtual member создан → учитывается в генерации меню |

## Сценарий C: Родители пожилые

Аналогично B (virtual) или A (если есть Telegram и телефон).

# 4. Разные предпочтения

| Ситуация | Как работает V1 |
|----------|-----------------|
| Один любит рыбу, другой — нет | `disliked_foods` в профиле → AI исключает при генерации |
| Разные цели (похудение / поддержание) | `nutrition_goal` per member; меню — компромисс с приоритетом admin |
| Ребёнок ест проще | virtual profile с age_months → упрощённые блюда в слоте «ужин» |
| Конфликт аллергий | Генерация не предлагает блюдо с аллергеном; замена предлагает safe alternative |

**Принцип:** пользователь не настраивает это ежедневно — настроил один раз в Профиле → AI применяет автоматически.

---

# 5. Аллергии

| Шаг | Где |
|-----|-----|
| 1 | Профиль → Питание / Ограничения (per member) |
| 2 | Выбор из списка + custom allergies |
| 3 | При генерации меню — hard exclude |
| 4 | При замене блюда — фильтр каталога |
| 5 | При добавлении в покупки — warning если конфликт (P1) |

**Семейный эффект:** один ребёнок с аллергией на орехи → всё семейное меню без орехов, не только «детская тарелка».

# 6. Общий список покупок (единый семейный центр)

PlanAm — **не только** генератор продуктов из меню. Это **единый список всех покупок семьи**.

| Аспект | V1 |
|--------|-----|
| Scope | `family_id` в shopping list API |
| Состав списка | Ингредиенты из меню + ручные покупки любого типа |
| Кто добавляет | Любой adult (ручной ввод); из меню — автоматически |
| Кто покупает | Любой; sync при открытии списка |
| Тип покупки | `recipe` · `home` · `child` · `pet` (AI или явный выбор) |
| Категории | Единые **14 канонических** (вкл. быт, дети, питомцы) |
| На главной | Счётчик «N товаров» агрегирует **весь** family list |
| Уведомление | «Список обновлён» → супругу (P1) |

Пользователь **не использует** отдельное приложение для бытовых покупок.

## 6.1. Сценарии семейных покупок

### Супруг добавил порошок

| Шаг | Действие |
|-----|----------|
| 1 | Супруг → Покупки → «+ Добавить» |
| 2 | Вводит «порошок» (или выбирает из подсказок) |
| 3 | AI: тип `home` → категория **🧻 Быт и уборка** |
| 4 | Позиция в общем списке; admin видит на ПланАм «8 товаров» |
| 5 | В магазине — отмечает купленным; sync всей семье |

### Родитель добавил подгузники

| Шаг | Действие |
|-----|----------|
| 1 | Admin/бабушка → Покупки → «+ Добавить» |
| 2 | «Подгузники размер 4» |
| 3 | AI: тип `child` → **🧸 Детские товары** |
| 4 | Не смешивается с продуктами меню; видно в своей категории |
| 5 | Супруг в магазине берёт и меню-ингредиенты, и подгузники из одного списка |

### Ребёнку нужна смесь

| Шаг | Действие |
|-----|----------|
| 1 | Admin замечает на ПланАм или добавляет вручную |
| 2 | «Детская смесь Nutrilon» |
| 3 | AI: тип `child` → **🧸 Детские товары** |
| 4 | Опционально: привязка к virtual child member (P1) |
| 5 | Уведомление супругу: «добавлено в список» (P1) |

### Закончился корм питомца

| Шаг | Действие |
|-----|----------|
| 1 | Любой adult → Покупки → «+ Добавить» |
| 2 | «Корм для кошки 2 кг» |
| 3 | AI: тип `pet` → **🐾 Для питомцев** |
| 4 | В списке рядом с продуктами меню — один поход в магазин |
| 5 | Счётчик на ПланАм включает и корм, и ужин |

# 7. Общее меню

| Аспект | V1 |
|--------|-----|
| Генерация | `POST /menus/generate` с family context |
| Участники | Все members + virtual nutrition |
| Просмотр | Все видят `/plan/today` одинаково |
| Замена | Admin или готовящий; AI учитывает всех |
| Порции | `servings` per slot; outcome sheet — per meal |

---

# 8. Уведомления семьи

| Тип | Получатель | Триггер |
|-----|------------|---------|
| Меню готово | Все adults | После генерации |
| Список покупок | Тот, кто обычно покупает | Новые позиции из меню **или ручного добавления** |
| Напоминание готовки | Готовящий | За N часов до ужина |
| Остатки истекают | Admin | pantry expiring |
| Приглашение | Invite target | Telegram bot |

Настройка: Профиль → Уведомления (per user, не per family).

---

# 9. Personal vs Family mode

| Режим | Когда | Что видит |
|-------|-------|-----------|
| **Personal** | Нет семьи или выбран личный | Меню и покупки на 1 человека |
| **Family** | В семье, family mode active | Общие меню, покупки, статусы |

Переключение: Профиль → Я (ProfileModeControl) — только если пользователь в семье.

---

# 10. Почему вместе удобнее (сводка)

| # | Причина |
|---|---------|
| 1 | Один список покупок (еда + быт + дети + питомцы) — не «скинь что купить» |
| 2 | Одно меню — не «что будем есть» каждый вечер |
| 3 | Аллергии ребёнка учтены для всех |
| 4 | Супруг видит тот же план без объяснений |
| 5 | Остатки общие — меньше еды в мусор |
| 6 | Здоровье семьи — одна картина на ПланАм |
| 7 | Приглашение за 30 секунд через Telegram |
| 8 | Virtual children — без отдельного приложения для малыша |
