# Причина коррекции

После согласования Product Master выявлено ограничение: раздел «Покупки» был описан преимущественно как **список ингредиентов из меню**.

Для V1 этого недостаточно. PlanAm должен быть **единым семейным списком покупок** — еда, быт, дети, питомцы в одном месте.

---

# Что изменено

## Продуктовый принцип (новый)

**PlanAm — единый семейный центр покупок**, а не только генератор продуктовых списков из меню.

Список включает:

```text
Ингредиенты из меню
Ручные покупки
Домашние товары
Товары для детей
Товары для животных
```

Пользователь не должен использовать отдельное приложение для бытовых покупок.

## Категории V1

**Было:** 13 food-категорий (овощи… заморозка… другое).

**Стало:** **14 категорий:**

| # | Категория |
|---|-----------|
| 1 | 🥬 Овощи и зелень |
| 2 | 🍎 Фрукты и ягоды |
| 3 | 🥩 Мясо и птица |
| 4 | 🐟 Рыба и морепродукты |
| 5 | 🥛 Молочные продукты |
| 6 | 🥚 Яйца |
| 7 | 🍞 Хлеб и выпечка |
| 8 | 🍚 Крупы, макароны и бакалея |
| 9 | 🧂 Специи, соусы и масла |

**Убрано из канона:** отдельные «Бакалея», «Заморозка», «Рыба» без морепродуктов — сведены в новые названия.

**Запрещено:** категория **«Продукты»** — не использовать ни в UI, ни в данных V1.

## AI-логика (новое)

AI различает **тип покупки**:

| Тип | Примеры | Категория |
|-----|---------|-----------|
| Покупка для рецепта | Курица, молоко | Мясо и птица · Молочные… |
| Покупка для дома | Порошок | Быт и уборка |
| Покупка для ребёнка | Подгузники, смесь | Детские товары |
| Покупка для питомца | Корм для кошки | Для питомцев |

# Как это влияет на V1

## Продукт

| До | После |
|----|-------|
| Покупки = производная меню | Покупки = **главный семейный список** |
| Только продукты питания | Еда + быт + дети + питомцы |
| Счётчик «N товаров» = ингредиенты | Счётчик = **все** позиции семьи |
| Отдельное app для быта | Не нужно |

## UX экрана Покупки

- Обязательная кнопка **«+ Добавить»** (P0)  
- Группировка по 14 категориям (включая 🧻 🧸 🐾)  
- Позиции из меню и ручные — в одном списке, визуально можно различать по `source` (P1)

## AI

- При ручном вводе: классификация типа + категории  
- При sync из меню: тип `recipe` по умолчанию  
- «Спросить PlanAm»: «добавь порошок в список» → категория Быт

## Семья

- Сильнее аргумент «почему вместе»: супруг добавляет быт, admin — детское, оба видят один список  
- Уведомление «список обновлён» срабатывает и на ручные добавления

## V2 доставка

- Единый список упрощает будущий checkout: один cart на family  
- Категории 🧻🧸🐾 потребуют retailer adapter для non-food SKU (P2/V2)

## Релиз (P0 блокеры)

Без **P0-SHOP-04..06** релиз с новой моделью **неполный**: пользователь не сможет вести бытовые покупки в PlanAm.
