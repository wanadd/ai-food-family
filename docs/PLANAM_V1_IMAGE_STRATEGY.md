# 1. Роль изображений

Фотографии — **главный визуальный элемент** продукта PlanAm.

**Почему:**

- Еда вызывает эмоцию и желание готовить — это часть ценности «готовое решение».
- Фото отвечает на «что готовить» быстрее текста (3-секундный тест).
- Визуал отличает PlanAm от «списка функций» и CRM-интерфейсов.
- Фото важнее декоративных карточек и служебных блоков.

**Принцип качества:**

- Фото должны вызывать **желание готовить**.
- Фото являются **частью ценности продукта**, не украшением.
- Фото **важнее** декоративных карточек.

# 2. Hero-фотографии (ПланАм `/`)

## Размер и место

| Параметр | V1 |
|----------|-----|
| Доля экрана | До **40%** высоты viewport (без скролла) |
| Позиция | Центр Hero, под приветствием, над CTA |
| Соотношение сторон | **16:9** или **4:3** (crop center) |
| Минимальная высота | ~140px на iPhone SE |

## Когда используются

Фото блюда в Hero — **только** когда Hero показывает **следующий приём пищи** (default happy path).

## Когда не используются

| Состояние Hero | Вместо фото |
|----------------|-------------|
| Нет меню | Иллюстрация «создать меню» + иконка |
| Критичные покупки | Иконка 🛒 / иллюстрация списка |
| Инсайт здоровья | Иконка ❤️ / компактный блок без еды |
| Ошибка загрузки | Placeholder (см. §6) |
| Первый запуск (до меню) | Onboarding-иллюстрация |

## Светлая тема

- Фото: полноцветное, яркое, без тяжёлых overlay.
- Скругление: **16–20px** (UI 2026).
- Лёгкая тень или тонкая рамка для отделения от фона.
- Текст поверх фото **не используется** — название и ккал под фото.

## Тёмная тема

- Те же фото; без инверсии.
- Фон карточки Hero: elevated surface (тёмно-серый).
- Скругление и пропорции — как в светлой.
- При отсутствии фото: muted иллюстрация на тёмном surface, не белый placeholder.

# 3. Когда показывать фото еды

| Экран / контекст | Фото | Размер |
|------------------|------|--------|
| **ПланАм** — Hero = следующий приём | Да | Крупное (§2) |
| **Сегодня** — карточка блюда | Да | Среднее; ужин — крупнее завтрака/обеда |
| **Сегодня** — главная карточка (ужин) | Да | Крупнее остальных слотов |
| **Рецепт** `/plan/recipes/[id]` | Да | Hero рецепта, полная ширина |
| **Каталог** | Да | Thumbnail в списке |
| **Замена блюда** (replace flow) | Да | Thumbnail в вариантах |

---

# 4. Когда НЕ показывать фото еды

| Контекст | Визуал |
|----------|--------|
| Нет меню (ПланАм, Сегодня empty) | Иллюстрация + короткий текст |
| Нет покупок (всё куплено) | Иконка ✓, без еды |
| Покупки — Hero «N товаров» | Иконка 🛒 / иллюстрация корзины |
| Пустые остатки | Иконка 📦 |
| Ошибка API / сеть | Иконка предупреждения + «Повторить» |
| Здоровье — инсайт Hero | Иконка / мини-график, не еда |
| Профиль, настройки, подписка | Аватар / иконки разделов, не еда |
| Покупки — список по категориям | Emoji категории, не фото еды |

### Placeholder (нет фото рецепта)

```text
[Скруглённый блок]
  Иконка блюда (outline)
  Название рецепта крупно
  Нейтральный градиент (тёплый, не серый амин)
```

Не показывать broken image icon.

# 5. Источник изображений

| Источник | Приоритет |
|----------|-----------|
| `recipe.image_url` из API / каталога | Primary |
| CDN / storage привязанный к `recipe_id` | Primary |
| Stock food library (internal) | Fallback по категории блюда |
| AI-generated (будущее) | P2, не V1 blocker |

## Если фото отсутствует

1. Попытка категорийный stock (суп → stock soup, салат → stock salad).
2. Если нет match — **placeholder** (§4), не пустота.
3. Никогда не блокировать экран из-за отсутствия фото.

## Fallback визуально

- Тот же размер и скругление, что у фото.
- Тёплый нейтральный градиент + иконка + название блюда.
- В каталоге — инициалы или emoji типа блюда (🍲 🥗 🍖) как крайний fallback.

# 6. Иерархия визуала (напоминание)

```text
Фото блюда (когда уместно)
  ↓
Карточка / иллюстрация действия
  ↓
Текст и CTA
```

**Не наоборот:** текстовый блок с мелкой иконкой не заменяет фото на food-экранах.

---

# 7. Связь с динамическим Hero

```text
Следующий приём пищи     → фото блюда
Нет меню                 → иллюстрация
Критичные покупки        → иконка списка
Инсайт здоровья          → иконка / инсайт-блок
Всё в порядке            → фото следующего приёма
```

Реализация: один Hero-компонент, prop `variant: meal | menu-empty | shopping | wellness | placeholder`.

---

# 8. Критерии приёмки (Sprint 1)

- [ ] ПланАм с меню показывает фото ближайшего приёма (не фиксированный ужин).
- [ ] ПланАм без меню — без food-фото, есть иллюстрация.
- [ ] ПланАм с shopping-hero — без food-фото.
- [ ] Рецепт и Сегодня — фото или placeholder, не broken image.
- [ ] Светлая и тёмная тема — без регрессий контраста.
- [ ] Fallback одинаковый размер с фото (нет layout shift).
