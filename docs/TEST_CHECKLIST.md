# Чек-лист ручного тестирования ПланАм

**Этап 12 — стабилизация.** Новые функции не добавляем; проходим приложение глазами пользователя и фиксируем дефекты в [BUGS.md](./BUGS.md).

**Окружение:** production или staging с реальным Telegram-ботом и Mini App.  
**Локально:** `npm run dev` + `ENVIRONMENT=development` на API → браузер на `http://localhost:3000` (dev-пользователь, см. баннер «DEV режим»).  
**Устройство:** эмулятор или телефон **360px** ширины (Mini App).  
**Роли:** обычный пользователь + владелец (`ADMIN_TELEGRAM_IDS`) для пунктов 23–25.

Обозначения: `[ ]` — не проверено, `[x]` — OK, `[!]` — баг (номер в BUGS.md).
