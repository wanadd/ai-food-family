# Принцип

Экран ПланАм — **один layout**, меняются **данные и CTA**, не структура. Пользователь всегда видит ответ на «что самое важное сейчас».

## Динамический Hero (Final Vision)

Hero **не привязан к ужину**. Показывает **следующий наиболее полезный шаг** по приоритету:

1. Нет меню → «Создать меню»
2. Ближайший приём пищи → фото + «Приготовить» (08:00 завтрак · 13:00 обед · 18:00 ужин)
3. Блюдо приготовлено → следующий приём
4. Покупки важнее → «N товаров» + «Открыть список»
5. Инсайт здоровья → «Не хватает белка» + «Подробнее»
6. Всё в порядке → следующий приём пищи

**Визуал Hero:** фото еды — только когда Hero = следующий приём пищи. Иначе — иллюстрация / иконка (см. Image Strategy).

**Базовая структура (неизменна):**

```text
Приветствие
[Hero: динамический — см. приоритет выше]
[Primary CTA]
━━━━━━━━━━
🛒 Купить      · статус
📦 Остатки     · статус
❤️ Здоровье    · статус
━━━━━━━━━━
[Вторичные CTA]
✨ Спросить PlanAm
```

# 1. Первый запуск

| Элемент | Содержание |
|---------|------------|
| Hero | Placeholder: «Давайте составим первое меню» |
| Primary CTA | **Составить меню** → `/plan/generate` или onboarding |
| 🛒 | «Пока пусто» |
| 📦 | «Добавим после готовки» |
| ❤️ | «Заполни профиль питания» → `/account/nutrition` |
| Вторичные | Скрыты или «Начать» |
| AI | «Спросить PlanAm» → подсказка «Сначала меню» |

**Цель:** за 1 экран понятно — следующий шаг = меню.
