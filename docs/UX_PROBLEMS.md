# UX-аудит ПланАм

Реестр UX-проблем по каждому разделу с указанием приоритета. Источник — текущая реализация (без изменения кода).

## Уровни приоритета

- **Критично** — блокирует ключевой сценарий или приводит к потере данных/денег.
- **Высокий** — путает пользователя, провоцирует отказ от продукта.
- **Средний** — снижает скорость работы, требует обучения.
- **Низкий** — косметические/мелкие.

## 1. Дублирующиеся экраны

| # | Название | Описание | Почему проблема | Влияние | Приоритет |
|---|---|---|---|---|---|
| 1.1 | `/settings/care` ≡ `/nutritionist/care` | Полный дубль одной страницы под двумя URL | Пользователь не понимает, какая «настоящая» | Запутывает после случайного попадания | Высокий |
| 1.2 | `/menu/recipes` → `/recipes` | Server redirect на тот же каталог | URL мигает в Telegram WebView | Лёгкое замешательство | Низкий |
| 1.3 | `/settings/units`, `/privacy`, `/language` | Заглушки на одном `SettingsPlaceholder` | Создают иллюзию функций, реально ничего не делают | Усталость, потеря доверия | Средний |
| 1.4 | «План на сегодня» на `/` и `/menu` | Один и тот же блок в двух местах с разным форматом | Расхождения при multi-day, разная типографика | Сомнения в данных | Высокий |
| 1.5 | `/menu/event` — недоступный экран | Существует, но ссылок в UI нет | Мёртвая функция; страница недоступна | Скрытая полу-готовая фича | Средний |

## 11. Запутанные сценарии

| # | Сценарий | Описание | Приоритет |
|---|---|---|---|
| 11.1 | Generate ↔ override ↔ choose | `wizardGoal` инициализируется из профиля, `personsCount` — из `localStorage`; после изменения профиля в другой вкладке состояние мастера не обновляется | Высокий |
| 11.2 | Care vs Notifications | Две раздельные модели «уведомлений» на разных URL | Высокий |
| 11.3 | Quick-action vs Replace | Один и тот же экшен доступен из 3 мест с разным flow | Средний |
| 11.4 | Member-edit | Профиль участника редактируется в `/family` (форма), а свой профиль — в `/profile/nutrition` (другой UI) | Высокий |
| 11.5 | `returnTo` цепочки | После `/nutritionist → /progress → /profile` `returnTo=/nutritionist` теряется | Средний |
| 11.6 | Pending в боте | Текст/voice/чек используют разные FSM-сущности; пользователь нажал не ту кнопку → нет отмены | Средний |
| 11.7 | Deep-link `/menu/current?replace=1` | Открывается модалка без явного индикатора, что это quick-action | Средний |
| 11.8 | Сохранение режима плана | `localStorage` vs ответ сервера в `MenuOverview.settings_summary` — могут разойтись после смены устройства | Средний |
