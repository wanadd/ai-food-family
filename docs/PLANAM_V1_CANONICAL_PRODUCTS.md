## Canonical products

- Все **234** уникальных названия каталога покрыты (0 → `другое`).
- Категории — из `categories_v1` (`овощи_зелень`, `мясо_птица`, `специи_соусы`, …).
- **Неоднозначные семейства разнесены по смыслу:** `перец чёрный/душистый/белый/кайен/чили` → `специи_соусы`; `перец болгарский/сладкий` → `овощи_зелень`. Это и есть корректное разрешение «ambiguous families».
- **Общие названия** (`овощи`, `специи`, `зелень`, `мясо`, `рыба`, `соус`, `грибы`, …) получают категорию, но помечаются `generic=true`, `nutrition_ready=false`.

## Единицы измерения

Canonical: `г, кг, мл, л, шт, ст.л., ч.л., стакан, зубчик, щепотка, упаковка`.

Нормализация по `UNIT_ALIASES`: `гр/грамм → г`, `ложка → ст.л.`, `шт. → шт`,
`зуб. → зубчик`, `пакетик/пачка/банка → упаковка`, и т.д. Мусор вида
`...4 ст.л.` (количество затекло в единицу) распознаётся извлечением известного
unit-токена.

## Количества

`по вкусу / немного / щепотка / 0 / пусто` → `to_taste=true`, число **не
выдумывается**. Валидные числа приводятся (`1,5 → 1.5`). Прочее остаётся как
есть и попадает в раздел «требует ручного решения».

---

## Что делает `--commit` (безопасный набор)

Только: `category` (canonical shopping slug), `unit` (canonical), числовое
`quantity` (реформат `,`→`.`). Идемпотентно (повторный прогон = 0 изменений).

**НЕ меняется автоматически** (выносится в отчёт для решения человеком):
- названия ингредиентов (вариантов нет — переименовывать нечего);
- `to_taste` количества (нужен отдельный флаг/колонка, иначе потеря данных);
- строки с категорией `другое` / generic — `needs_review`.

> `recipe_ingredients.category` обновляется. Денормализованная копия
> `recipes.ingredients` (JSONB) в этом этапе не трогается — её ресинк вынесен в
> следующий шаг, чтобы не расширять scope.

## Запуск

```bash
# DRY-RUN (по умолчанию) — отчёты в reports/, БД не меняется
python backend/scripts/normalize_recipe_ingredients.py --dry-run

# COMMIT — применить безопасный набор (идемпотентно)
python backend/scripts/normalize_recipe_ingredients.py --commit
```

В api-контейнере на сервере:

```bash
docker compose -f docker-compose.prod.yml exec api sh -lc \
  "cd /app && python backend/scripts/normalize_recipe_ingredients.py --dry-run"
```

Выход:
- `reports/planam_v1_ingredient_normalization_dry_run.md`
- `reports/planam_v1_ingredient_normalization.json`
