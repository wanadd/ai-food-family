# PLANAM V1 — Final Vision

**Версия:** 1.1 (freeze)  
**Дата:** 2026-06-03  
**Статус:** **зафиксирован** для V1 — изменения только через явное согласование  
**Приоритет:** **высший** — обязателен к прочтению перед любой разработкой

Если реализация противоречит данному документу — реализация считается **неправильной**.

---

# ЧТО ТАКОЕ PLANAM

PlanAm — семейный AI-помощник по питанию, меню и покупкам.

PlanAm **не является:**

- каталогом рецептов;
- трекером калорий;
- списком покупок;
- AI-чатом.

Все эти функции являются **инструментами**.

---

Настоящая цель PlanAm:

```text
Снять ежедневную нагрузку принятия решений о питании семьи.
```

---

# ГЛАВНЫЙ ВОПРОС ПРОДУКТА

После открытия приложения пользователь **не должен** думать:

```text
Что нажать?
Где нужный раздел?
Что сегодня готовить?
Что покупать?
```

Пользователь должен **сразу получить ответ**.

---

# ОСНОВНАЯ ЦЕННОСТЬ

PlanAm не продаёт функции.

PlanAm продаёт:

```text
Готовое решение.
```

---

# ЧЕТЫРЕ ГЛАВНЫХ ВОПРОСА

Любой экран должен помогать ответить минимум на один из вопросов:

```text
Что готовить?

Что купить?

Что есть дома?

Что важно по здоровью?
```

Если экран не помогает ответить ни на один из этих вопросов, он **не относится к первому уровню** продукта.

---

# РОЛЬ AI

AI существует для **снятия решений**.

- AI не является самостоятельным разделом приложения.
- AI не должен быть главной вкладкой.
- AI не должен заставлять пользователя писать запросы каждый день.

Пользователь должен видеть **результат работы AI**, а не сам AI.

**Плохо:**

```text
Открой AI
Напиши запрос
Спроси модель
```

**Хорошо:**

```text
Вот меню на неделю.

Вот список покупок.

Вот лучший вариант ужина.

Вот рекомендация по здоровью.
```

# HERO

Hero — **главный объект** экрана.

- Hero всегда занимает центральное место.
- Hero **НЕ** привязан к ужину.
- Hero **НЕ** привязан к конкретному блюду.
- Hero является **динамическим**.

---

## Главное правило Hero

Hero показывает **следующий наиболее полезный шаг** для семьи.

### Приоритет

#### 1. Нет меню

```text
Составим меню?

[Создать меню]
```

#### 2. Приближается ближайший приём пищи

```text
Следующий приём пищи

Фото блюда
Название
Время
Калории

[Приготовить]
```

Пример по времени суток:

- 08:00 → завтрак
- 13:00 → обед
- 18:00 → ужин

#### 3. Блюдо приготовлено

Показать **следующий** приём пищи.

#### 4. Покупки важнее

```text
Нужно купить 12 товаров

[Открыть список]
```

#### 5. Важный инсайт здоровья

```text
Сегодня не хватает белка

[Подробнее]
```

#### 6. Всё в порядке

Показать следующий приём пищи.

---

## Главный визуальный объект Hero

**По умолчанию** главный визуальный объект — **еда** (фото блюда).

**Исключение:** если для семьи существует **более важное действие**, Hero перестраивается под это действие — без фото еды.

| Ситуация | Hero | Визуал |
|----------|------|--------|
| Нет меню | Создать меню | Иллюстрация / иконка, не фото блюда |
| Критически важные покупки | Открыть список | Иконка 🛒 / иллюстрация списка |
| Важный инсайт здоровья | Подробнее | Иконка ❤️ / компактный инсайт |
| В остальных случаях | Следующий приём пищи | **Фото блюда** |

Детали: [PLANAM_V1_IMAGE_STRATEGY.md](./PLANAM_V1_IMAGE_STRATEGY.md).

# СЕМЬЯ

Семья является одним из главных преимуществ продукта.

PlanAm должен работать **лучше для семьи**, чем для одного человека.

Основные преимущества:

```text
Общее меню
Общий список покупок
Общие остатки
Общие уведомления
```

---

# ВИРУСНОСТЬ

Рост строится через **семью**.

Не через геймификацию. Не через баллы. Не через достижения.

Пользователь приглашает супруга, родителей, детей и других членов семьи — потому что это делает продукт **полезнее**.

---

# ВИЗУАЛЬНЫЙ ЯЗЫК

Референс используется как источник **принципов**, не как шаблон для копирования.

**Берём:**

```text
Воздух

Простоту
Эмоции

```

**Не берём:**

```text
Копирование экранов
Копирование элементов
Копирование структуры
```

## Главный визуальный объект (продукт)
