## Условные обозначения

- `Частота` — оценка использования: 🔴 Очень часто, 🟡 Часто, 🟢 Иногда, ⚪ Редко.
- Колонки «Оставить / Объединить / Удалить» — рекомендация для ближайшего рефакторинга, без потери функциональности.

## 2. Лишние экраны

1. `/menu/recipes` — server redirect, можно удалить, перевязав ссылки на `/recipes`.
2. `/settings/units`, `/settings/privacy`, `/settings/language` — заглушки.
3. `/settings/care` — полный дубль `/nutritionist/care`.
4. `/menu/event` — скрытая функция, не используется.
5. `/menu/settings` — override-параметры можно вынести в шапку мастера.

---

## 3. Дублирующиеся экраны

1. `/settings/care` = `/nutritionist/care`.
2. `/menu/recipes` = `/recipes` (redirect).
3. `/settings/about` ≈ `/settings/support` — почти один и тот же объём контента; стоит объединить в `/settings/about`.
4. Часть `/profile` дублирует `/settings` (Уведомления, О приложении).
5. Onboarding-шаги `goals/diets/allergies/restrictions` дублируются с `/profile/nutrition`.
6. Чекины (`MealCheckinPanel`) частично дублируют функционал `meal_leftovers`.
7. Quick-action «Заменить блюдо» (`/menu`) дублирует кнопку в карточке `/menu/current`.

---

## 4. Самые сложные сценарии

1. **Создание меню** — мастер 5 шагов, чеклист, лимиты, Амы, выбор варианта, активация плана (`USER_FLOWS §5`).
2. **Приглашение участника** — три пути (по номеру, invite-link, виртуальный), каждый со своими согласиями (`§4`).
3. **Голосовое добавление** — распознавание + AI parse + pending-список (`§10`).
4. **OCR чека** — распознавание + сопоставление с категориями + сохранение в пантри/покупки.
5. **Замена блюда** — модальный flow + контекст AI + списание Амов + обновление плана (`§7`).
6. **Care + notifications** — два независимых раздела уведомлений (`§19`).
7. **Family-billing** — distribute Амов и тарифа на нескольких пользователей (`PRODUCT_LOGIC §11–12`).
8. **Прогресс к цели** — параллельные расчёты в backend и frontend.

## 5. Самые важные сценарии

1. **Создание меню** — основная ценность.
2. **Использование плана** (просмотр текущего, чекины, замены) — ежедневное.
3. **Покупки** — действие, ради которого нужны меню/запасы.
4. **Запасы и остатки** — экономия и предотвращение дубля покупок.
5. **Подписка и Амы** — монетизация.
6. **Регистрация** — без неё ничего не работает.
7. **Telegram-bot quick add** — отдельный канал для занятых.

## 6. Бесполезные кнопки

1. «Купить Амы» — disabled.
2. Onboarding `favoriteFoods` / `dislikedFoods` («Далее») — данные сохраняются, не используются нигде.
3. «Подтверждаю право управлять» при создании семьи — стейт без записи в БД.
4. Шесть quick-actions в `/menu` (`cheaper`, `use_pantry`, `more_protein`, `less_time`) — текстовый ответ без реального изменения меню.
5. «Улучшить / Совместимость / Оценить» в `RecipeDetailModal` без Амов/AI — молчат.
6. `quick:voice_hint` / `quick:receipt_hint` / `quick:shopping_hint` / `quick:pantry_hint` — только текстовые подсказки.
7. «Скоро» в заглушках настроек.
8. Дубликаты CTA (см. `BUTTON_MAP §27`).

## 9. Экраны для переработки в первую очередь

## 10. Проблемы, мешающие массовому запуску

1. **Отсутствие реальной оплаты** — `/subscription` без подключённого provider'а, выбор тарифа без списания.
2. **Care и notifications — две модели** — пользователь не понимает, где включить какой тип уведомлений.
3. **Замена блюда и AI-функции списывают Амы без предупреждения** — нарушение прозрачности биллинга.
4. **OCR / voice / chat не работают без Амов** — без подсказки и CTA «купить Амы».
5. **iOS Telegram WebView блокирует `.ics`-скачивание** — функция «Добавить в календарь» не работает на iOS.
6. **Профиль доступен только с главной** — пользователь теряется.
7. **Multi-day обзор показывает только день 1** — в обзоре `/menu` нет понимания, что план на неделю.
8. **Care уведомления могут приходить ночью** при пустых quiet-hours.
9. **Дублирующие модели нутрициологии для virtual и реальных участников** — расхождения в советах.
10. **Заглушки в настройках (units / privacy / language)** — снижают доверие.
11. **Сценарии бота для voice/photo при отсутствии Амов** дают сообщение об ошибке только после транскрипции (трата ресурсов).
12. **Pending-список бота** при partial-failure сохраняет часть позиций — отсутствует rollback.
13. **`/menu/event` существует как мёртвая функция** — пользователь может попасть туда по URL.
14. **Поля онбординга `favoriteFoods` / `dislikedFoods` не отображаются после** — обман ожиданий.
15. **Расхождение расчёта прогресса** между backend и frontend.

## 11. Идеальная структура ПланАм (без потери функциональности)

```
Главная (/)
├── Меню (/menu)
│   ├── Мастер генерации (/menu/generate)
│   │     [бюджет+режим объединены, override параметров — в шапке]
│   ├── Активный план (/menu/current)
│   │     [чекины + замены + остатки в одном экране]
│   └── Каталог рецептов (/recipes)
│         └── Карточка (/recipes/[id])
│
├── Покупки (/shopping)
│   └── Категории / добавление через единый sheet
│
├── Запасы (/pantry)
│   └── Inline-форма и фильтр «скоро заканчиваются»
│
├── Нутрициолог (/nutritionist)
│   ├── Сегодня (компактный KPI)
│   ├── Прогресс к цели (раскрываемая секция)
│   ├── Совет + Отложенные (один список)
│   └── Чат (/nutritionist/chat)
│
└── Профиль (/profile) [доступен также из нижней панели]
    ├── Питание (/profile/nutrition)            ← разбит на этапы
    ├── Семья (/family)                          ← объединённые sheet'ы
    ├── Прогресс (/progress)                     ← журнал тренировок и веса
    ├── Подписка (/subscription)                 ← реальный оплат
    ├── Уведомления (/notifications)             ← единая модель push (включая care)
    ├── Документы (/settings/documents)
    ├── Аккаунт (/settings/account)
    └── О приложении (/settings/about)           ← + support
```

**Что объединено**

- `/nutritionist/care` + `/notifications` → одна страница «Уведомления».
- `/menu/settings` → шапка `/menu/generate`.
- `/menu/event` → опциональный шаг `Тип плана` в мастере (или удалить).
- `/menu/recipes` → удалить.
- `/settings/units` / `/privacy` / `/language` → удалить, перенести содержимое (когда появится) в Документы/Аккаунт.
- `/settings/support` → объединить с `/settings/about`.
- Виртуальные участники и реальные — один универсальный профиль (через флаг `is_virtual`).

**Что добавлено как навигация**

- Кнопка «Профиль» в нижней панели (вместо «ПланАм/Дом» отдельный таб, либо контекстная замена).
- На главной — CTA «Дозаполни профиль» / «Активируй подписку», если что-то нужно действие.

## 12. Краткая сводка результатов аудита

- **Экранов найдено**: 41 уникальный маршрут (web) + 7 модальных листов/полноэкранных форм + 5 типов inline-форм. Всего «экран-эквивалентов»: 53.
- **Роутов найдено**: 41 (Next.js App Router) + 0 динамических `[slug]` (только `[id]` в `recipes`, `admin/users`, `admin/families`).
- **Кнопок найдено**: ≈360 (web + sheets + bot), из них дублирующих ≈22, без обработчиков/disabled ≈18.
- **Пользовательских сценариев найдено**: 23 ключевых (см. `USER_FLOWS.md`).
- **UX-проблем найдено**: 96 (Критично 2, Высокий 26, Средний 44, Низкий 24).
- **Критические проблемы**:
  1. Замена блюда / AI-функции списывают Амы без предупреждения (`UX_PROBLEMS 14.8 / 20.7`).
  2. `/subscription` без оплаты, выбор тарифа меняет статус без подтверждения (`22.2 / 22.5`).
